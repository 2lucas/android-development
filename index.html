Using Android Studio

Master the most powerful integrated development environment for building Android applications.

LS
Lucas de Souza
December 14, 2025
5 min read

Now that you've installed Android Studio, let's start using it. In Android Studio, click on the Terminal and navigate to your desktop using the command below.

Terminal 
cd Desktop
Tip

The command above is just an example for navigating directories in the terminal. Always use commands that you recognize and understand.

Exploring the main tools

Android Studio offers a robust set of tools that facilitate the entire development cycle. Getting to know each of them will significantly speed up your workflow.

  • Project: view the entire project structure, including modules, folders, configuration files, and resources. It's your overview of everything that makes up the application.
  • Gradle: manage build tasks, sync dependencies, and configure app variants. Gradle is the heart of the Android build system.
  • Logcat: monitor real-time logs of your app running on the emulator or physical device. Essential for debugging and behavior analysis.
  • Terminal: access Git, run Gradle commands, execute local scripts, and perform diagnostics directly within the IDE.

Essential Terminal Commands

Before diving into development, it's essential to know the basic commands you'll use frequently in the terminal. Here are the most important ones:

Navigating directories:

Terminal 
cd folder-name       # Enter a folder
cd ..                # Go back one folder
cd ~                 # Go to home folder
pwd                  # Show current directory

Listing files and folders:

Terminal 
ls                   # List files and folders
ls -la               # List with details and hidden files
ls -lh               # List with readable sizes

Manipulating files:

Terminal 
mkdir new-folder     # Create a new folder
touch file.txt       # Create an empty file
rm file.txt          # Remove a file
rm -rf folder        # Remove folder and contents
cp source dest       # Copy file
mv source dest       # Move or rename

Gradle Commands in Android Studio

Gradle is Android's build system. Knowing its commands will give you full control over your app's compilation:

Terminal 
./gradlew build              # Build the project
./gradlew clean              # Clean build files
./gradlew assembleDebug      # Generate debug APK
./gradlew assembleRelease    # Generate release APK
./gradlew installDebug       # Install on device
./gradlew test               # Run tests
./gradlew tasks              # List all tasks
Important

On Windows, use gradlew.bat instead of ./gradlew. The ./ prefix is used on Unix systems (Mac/Linux).

Essential Git Commands

Version control is fundamental in any project. Here are the Git commands you'll use most often:

Terminal 
git init                     # Initialize a repository
git status                   # Check file status
git add .                    # Add all files
git commit -m "message"      # Make a commit
git push origin main         # Push to remote repository
git pull origin main         # Pull updates
git branch                   # List branches
git checkout -b new-branch   # Create and switch to new branch
git log --oneline            # Summarized history

ADB - Android Debug Bridge

ADB is a powerful tool that allows direct communication with Android devices. Here are the most useful commands:

Terminal 
adb devices                  # List connected devices
adb install app.apk         # Install an APK
adb uninstall com.app.name  # Uninstall an app
adb logcat                   # Show real-time logs
adb shell                    # Access device shell
adb push local remote        # Send file to device
adb pull remote local        # Download file from device
adb reboot                   # Restart the device

Android Studio Keyboard Shortcuts

Mastering shortcuts will double your productivity. Memorize these first:

  • Ctrl/Cmd + Shift + A: Find Action
  • Ctrl/Cmd + N: Search classes
  • Ctrl/Cmd + Shift + N: Search files
  • Ctrl/Cmd + B: Go to definition
  • Alt + Enter: Show suggestions and quick fixes
  • Ctrl/Cmd + /: Comment/uncomment line
  • Ctrl/Cmd + D: Duplicate line
  • Ctrl/Cmd + Shift + F: Search in entire project
  • Shift + Shift: Search everywhere
  • Ctrl/Cmd + Alt + L: Format code

Setting Up the Emulator

To test your apps, you need an emulator or physical device. Follow these steps to set up:

  • 1. Open AVD Manager (Tools ‚Üí AVD Manager)
  • 2. Click "Create Virtual Device"
  • 3. Choose a device model (Pixel 6 is a good option)
  • 4. Select a system image (recommended: latest Android version)
  • 5. Configure RAM and storage as needed
  • 6. Finish and start the emulator
Performance

For better emulator performance, enable hardware acceleration (Intel HAXM on Windows/Mac or KVM on Linux). This makes a huge difference in speed.

Basic Project Structure

When you create a new project, Android Studio generates a standard folder structure. Here's what each one means:

  • app/src/main/java: Kotlin/Java source code for your app
  • app/src/main/res: Resources (layouts, images, strings)
  • app/src/main/AndroidManifest.xml: Main app configuration
  • app/build.gradle: Module build settings
  • build.gradle: Project build settings
  • gradle.properties: Gradle properties
  • settings.gradle: Module configuration
